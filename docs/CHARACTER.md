## 1. 정체성 / 서사

| 항목 | 값 |
|---|---|
| **이름** | 헬린 (Helin) |
| **이름 어원** | 숲 모티프 가공명 + 라틴 어원(Helian/태양)의 부드러운 변형. 빛 능력과 사제 톤이 한 번에 잡힘 |
| **나이** | 18~22 (청년기) |
| **성격 키워드** | 차분함 / 사명감 / 내면의 의심 |

### 서사 위치 (STORY.md §3 요약)

제사장 가문의 후예. 선조를 숲을 지킨 영웅으로 알고 자랐으며, 그 뜻을 이어 저주를 해결하기 위해 숲에 진입한다. 물려받은 능력 두 가지:
- **시간 조작**: 멈춘 시간을 흐르게 하고 태양 각도를 조절
- **등불**: 빛을 만들어 그림자를 생성·조작

탐색 중 점차 불편한 진실을 마주한다 — 자신이 쓰는 능력이 선조가 그림자 왕가를 배신한 대가로 얻은 것이며, 그 선택이 선의가 아닌 이익에서 비롯됐음을 알게 됨.

### 호칭 변화 여지 (서사 장치)

후반 진실 발견 후 "이름이 의미를 잃는" 연출 — 예: 선조 기록·NPC가 "등불의 아이"로 호칭. 이름 자체는 부담 없되 후반 정서적 임팩트의 장치로 활용 가능. **잠정** — Phase 4-C/4-D STORY.md 갱신 시 확정.

## 2. 외형 실루엣

### 컨셉 골격

**상체 = 사제 (후드 + 로브 + 황금 데코) / 하체 = 기동성 (타이트 레깅스 + 단화) / 왼손 = 등불 고정 / 오른손 = 빛무기 (공격 순간만 응결)**

다크소울 마법사들이 자주 쓰는 "상체=정적·의식 / 하체=액션·기동" 이중 실루엣. 사제 정체성과 메트로배니아 액션성의 균형.

### 부위별 결정

| 부위 | 결정 | 비고 |
|---|---|---|
| **후드** | 반쯤 걸침 (이마~눈 위만 그림자) | 픽셀 64×64에서 표정 표현 가능 + 신비감 유지. 컷신(REC-FX-006)에서 후드 들어올리는 연출 가능 |
| **로브** | 짧은 깔끔 (허벅지 위~무릎 위) | 빠른 모션 표현 용이. "차분함" 성격과 일관된 정적 실루엣 |
| **하의** | 타이트 레깅스 + 단화 | 다리 라인 명확, 사제 톤과 액션 톤 콘트라스트. 부츠 대신 단화로 가벼운 인상 |
| **머리** | 어깨선 길이 | 묶음 여부(자연스럽게 풀린 / 부분 묶음)는 도트 시안 단계에서 결정 — 가변 |
| **옷차림 디테일** | 부분적으로 닳은 사제복 + 어깨 숄/스카프 가능 | "물려받은 것의 무게" 주제와 직결. 초반 깨끗 → 후반 닳음·핏자국 추가 여지 |
| **황금 데코** | 의상 가장자리·문양 단위로 한정 (가슴 한 줄·소매 끝·후드 가장자리 등) | 면적은 작게. 등불과의 시각 경쟁 회피 (§6 팔레트 가이드 참조) |

## 3. 등불 (캐릭터 시각적 핵심)

### 형태

| 항목 | 값 |
|---|---|
| **기본 형태** | 고전 랜턴 (육각/사각 케이지 + 손잡이 + 위쪽 캡) |
| **시각 톤 매칭** | 현재 저장소 [hud_consumable_hp.png](../assets/ui/hud/hud_consumable_hp.png) — 강한 외곽선 + 위쪽 어두운 캡 + 선명한 단색 몸체 구조 |
| **약간의 비틀기** | 베이스가 둥글거나 꼭대기에 작은 장식 첨탑 등으로 정형성 탈피. "Hollow Knight 클래식 + Blasphemous 종교적 디테일" 톤 |
| **크기** | 잠정 8~10px (실효 캐릭터 ~64px 기준 중간) — 도트 시안 작업 중 변경 가능 |

### 빛 색·심지

| 항목 | 값 |
|---|---|
| **외곽 빛** | 따뜻한 황금 (PointLight2D) |
| **코어** | 옅은 시안 |
| **심지** | 작은 빛 결정 (1~2px 시안 점) — "응축된 권능 조각" 시각화 |

**의도**: 외곽=황금(빛 능력) + 코어=시안(시간 능력). 선조가 받은 두 능력이 등불 하나에 시각적으로 동시 드러남. 후반 진실 발견 시 시안 코어가 강해지거나 보랏빛으로 흔들리는 연출 여지.

### 발광 매체 (서사 베이스)

**그림자 왕가의 봉인된 권능 조각**. 진실 발견 시 "내가 든 빛은 그들의 갇힌 힘이었다" 임팩트의 시각 베이스. STORY.md §3·§5 "물려받은 것의 무게"와 직결.

## 4. 빛무기 (오른손)

### 시스템 제약 (COMBAT.md §2 / §11)

- **무기 1종 = 검** (콤보 4타 횡베기, 강화 이동 후속 공격 4종 모두 검 베이스로 설계됨)
- 다른 무기 변형(창/낫 등)은 콤보·피니시·강화 이동 모두 재설계가 필요하므로 **본 컨셉에서는 검 1종 유지**
- 속성 차이는 **이펙트**(잔상/트레일/색상)로 표현 — 무기 형태는 통일

### 디자인 결정

| 항목 | 결정 |
|---|---|
| **검 실루엣** | 한손 세이버 / 커틀러스 (50~60px 길이, 살짝 휜 형태) — 횡베기 콤보 모션 정합 |
| **빛 질감** | 결정형 (단단한 코어 + 외곽 1~2px 산란 픽셀) — 등불 시안 결정 심지와 시각 언어 통일 |
| **평상시 색상** | 황금 단일 (등불과 통일) |
| **피니시 색 분기** | 코드가 자동 처리 — `EffectsSystem.get_finish_color` API. light=흰 강조 / shadow=보라 / hybrid=시안 |
| **등불-검 관계** | **공격 순간만 응결** — idle/run/jump/fall은 빈 오른손 또는 작은 빛 점만, slash_1~4는 검 응결한 상태 |

### 이점

- **컨셉 정합**: "등불 → 빛 응축 → 검" 흐름이 시각적으로 명확
- **작업 효율**: 모든 모션에 검을 그릴 필요 없음 (기본 상태는 빈 오른손) — 작업량 절감
- **서사 확장**: 후반 등불 빛이 보랏빛으로 흔들리면 검도 자동 톤 변화 (피니시 색 코드 분기 그대로 활용)

## 5. 비례 / 모션 톤

### 비례

**3.5등신 chibi 톤** — Death's Door / Eastward / Sea of Stars 류. 사제복·로브 같은 옷자락 표현이 머리 비례 덕에 명료해짐. 1구역 placeholder 스프라이트들과도 일관성 유지.

> 이후 모든 캐릭터/적/NPC 명세도 동일 비례 기본 적용. 보스나 특수 거대 적은 예외.

### 모션 의도

| 상태 | 의도 |
|---|---|
| **idle** | 숨쉬기 + 등불 흔들림. 정적이지 않은 미세 모션. 차분함 성격 반영 |
| **공격 (slash_1~4)** | 오른손에 빛 검 응결 + 횡베기. 1~3타는 묵직, 4타는 속성별 시각 분기 |
| **hurt** | 한 손 방어 자세 — 등불을 감싸는 듯한 인상. 등불 자체는 위치 일관 |
| **death** | 등불이 바닥에 떨어져 흔들리는 연출 — 서사적 중요 순간 |
| **컷신** | 후드를 들어올리는 연출 (REC-FX-006 정지 일러스트 컷신 인프라와 시너지) |

### 후반 시각 변화 여지

- **등불 색**: 황금+시안 코어 → 보랏빛 흔들림 (진실 발견 후)
- **의상**: 깨끗 → 닳음·핏자국 (스토리 진행)
- **빛 검**: 등불 색 변화에 따라 자동 톤 변화 (피니시 색 코드 분기)

## 6. 팔레트 (사용자 자율 결정)

> **결정 방식**: 사용자가 픽셀아트를 직접 제작하시므로 정확한 hex 값은 사용자 자율. 본 문서는 **유지해야 할 관계**만 명시. 시안 완성 시 실제 사용 색으로 본 섹션 갱신.

### 6.1 4가지 핵심 관계 (필수 준수)

1. **등불이 화면에서 가장 밝은 점**
   - 의상 황금 데코의 채도·명도 < 등불 황금
   - PointLight2D 부착 지점에 시선 집중

2. **얼굴 영역 식별 가능**
   - 후드 그림자 안에서도 피부·눈이 보일 만큼 피부 명도 충분
   - 머리 색이 후드 색과 너무 가까우면 이마/머리 라인이 묻힘 — 콘트라스트 확보

3. **명도 V자 구조 (권장)**
   - 후드(어둠) → 얼굴/등불(밝음) → 하의(어둠) — 시선이 중앙 상단(등불)에 집중

4. **속성 피니시 색은 코드가 담당**
   - 평상시 의상·검 색은 사용자 자유
   - 4타 피니시·강화 이동 잔상의 light/shadow/hybrid 색은 EffectsSystem이 동적 처리 (`get_finish_color` API)

### 6.2 잠정 참고값 (필수 아님 — 시작점)

> 사용자 시안 작업 시 위 4가지 관계만 점검하면 OK. 아래 hex는 안전한 시작점일 뿐 강제 아님.

| 영역 | 잠정 hex | 톤 |
|---|---|---|
| 후드 | `#4A3A55` | 다크 플럼 |
| 로브 | `#5A4A65` | 미디엄 플럼 |
| 언더 | `#E8DCC0` | 크림 |
| 레깅스 | `#2A2535` | 다크 차콜 보라 |
| 단화 | `#1F1A28` | 검은 보라 |
| 황금 데코 | `#C8A04A` | 머스타드 골드 (등불보다 채도 ↓) |
| 머리 | `#3A2818` | 다크 브라운 |
| 피부 | `#E8D5C5` | Cool Pale |
| 눈 | `#D4A030` | 호박색·앰버 |
| **(참조) 등불 외곽** | `#FFD27F` | 따뜻한 황금 — 화면 최밝점 |
| **(참조) 등불 코어** | `#7AC8C8` | 옅은 시안 |
| **(참조) 빛 검** | `#FFD27F` | 등불과 통일 (피니시 시 코드 분기) |

## 7. 3레이어 규약 (등불은 광원 대상)

[project_art_three_layer_rule.md 메모리 / 발광 스프라이트 3레이어 규약](../docs/recommendations/02_effects.md) 적용 — 등불은 광원 대상이므로 base/core/halo 3레이어 분리 제작 필수.

### 등불 3레이어 구조

| 레이어 | 내용 | 파일 |
|---|---|---|
| **base** | 등불의 메탈/케이지 윤곽 (어두운 외곽선 + 캡 등) | `lantern_base.png` |
| **core** | 황금 외곽 빛 + 시안 코어 + 결정 심지 | `lantern_core.png` |
| **halo** | 등불 주변 발광 그라데이션 (PointLight2D와 별개로 스프라이트 발광 표현) | `lantern_halo.png` |

Aseprite에서 등불을 별도 레이어로 두면 export 시 자동으로 3개 PNG 생성 가능. Godot 측에서는 3노드(Sprite2D base + core + halo)로 컴포지트.

### 캐릭터 본체 + 등불 분리 권장

캐릭터 본체와 등불은 별도 노드로 두는 것을 권장 — 등불에만 PointLight2D 부착 + halo 셰이더 적용. 단, idle 모션의 등불 흔들림은 캐릭터 모션과 맞추기 위해 부모-자식 관계로 묶음.
