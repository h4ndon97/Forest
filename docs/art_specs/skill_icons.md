## 3. 아이콘 — light_slash (빛 베기)

**모티프**: 북동→남서로 휘두른 초승달 검광.

### 기하
- 외곽 호 반경 7.5 px, 내측 호 반경 4.5 px (초승달 폭 3 px)
- 호 각도 -63° → 135° (북동 근처 → 남서 근처)
- 세그먼트 14개

### 레이어 (아래→위)
1. **본체 초승달** (`draw_colored_polygon`) — `#F2CC66` 금색
2. **선단 하이라이트** (외곽 0.8px 안쪽 1px 폴리라인) — `#FFF4CC` 아이보리
3. **잔광 점** 2개 (호 내측 중앙 양옆 ±0.18 rad, 반경 0.8 px) — `#C99A3F` 오커

### 픽셀아트 교체 팁
- 픽셀 해상도에서 초승달 두께는 최소 2 px 확보
- 선단 아이보리 1 px 점만으로 충분한 휘도 변화
- 잔광은 단일 1×1 점 2개가 가장 안정적

## 4. 아이콘 — shadow_strike (그림자 강타)

**모티프**: 중심에서 방사되는 6개 스파이크 + 2겹 코어.

### 기하
- 스파이크 6개, 12시부터 60° 간격
- 스파이크 길이 7.5 px, 밑변 반폭 1.6 px (삼각형)
- 중심 외곽 원 r=3.0, 하이라이트 원 r=1.7

### 레이어 (아래→위)
1. **스파이크 6개** (`draw_colored_polygon` 삼각형) — `#8B2FC6` 보라
2. **코어 외곽** — `#3A0D5C` 진보라
3. **코어 하이라이트** — `#C95AE8` 자홍

### 픽셀아트 교체 팁
- 픽셀에선 6방향 방사가 깨지기 쉬움. 4방향 + 대각 작은 점이 더 안정적
- 코어 2겹은 3×3 진보라 + 1×1 자홍으로 축약 가능

## 5. 아이콘 교체 워크플로

1. **PNG 제작**: 20×20 투명 배경, 중앙 정렬
2. **임포트 설정**: 필터 끔(Nearest), Mipmap 끔
3. **리소스 등록**: `data/skills/<skill_id>.tres`의 `icon_path`에 경로 지정
4. **자동 반영**: `skill_slot.gd._setup_icon_sprite`가 감지하여 Sprite2D로 렌더, 프로그래밍 아이콘은 자동 숨김

```
ResourceLoader.exists(icon_path) == true  →  외부 PNG 사용
                                  == false →  프로그래밍 폴백
```

## 7. 체크리스트 (교체 시)

- [ ] 아이콘 외곽이 MOSS_RADIUS(13px) 이내에 들어가는가
- [ ] 속성색 링이 아이콘을 가리지 않는가 (링은 r=15, 아이콘은 r≤10)
- [ ] 640×360 원본 해상도에서 식별 가능한가 (3배 확대 후 보정하지 말 것)
- [ ] 쿨다운 스윕(시계방향, 12시 시작)이 아이콘 위로 어둡게 덮이는가
- [ ] 키 라벨이 아이콘과 겹치지 않는가
