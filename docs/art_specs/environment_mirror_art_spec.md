# 시간이 멈춘 숲 — 거울(Mirror) 환경 오브젝트 아트 명세서

> 거울은 STOPPED(시간 정지) 중 플레이어가 [E]로 회전 각도를 조절할 수 있는 퍼즐/전투 환경 오브젝트다.
> FLOWING(시간 흐름) 진입 순간, 거울이 반사하는 부채꼴 영향권 내의 적을 "그림자 파편" 스포어로 분열시킨다.
> "시간이 멈춘 동안 정보를 정리하고, 흐르는 순간 한꺼번에 결과가 해제된다"는 메카닉을 시각적으로 드러내야 한다.
>
> **상위 문서**: [ui_design_master.md](ui_design_master.md) 섹션 D (월드 인터랙션 UI) 참조.
> 환경 오브젝트의 하이라이트/프롬프트 스타일은 상위 문서에서 D 카테고리 차원에서 통합 결정됨.

## 1. 캔버스 / 크기 규격

| 항목 | 값 | 비고 |
|---|---|---|
| **본체 캔버스 크기** | 64×64 | 여백 포함. 실효 본체 크기 24×40 |
| **실효 본체 크기** | 약 24×40 | 현재 마름모 Polygon2D 기준 (-12~+12 가로, -20~+20 세로) |
| **타일 대비** | 약 0.4×0.6 타일 | 바닥에 세워진 중소형 환경 오브젝트 |
| **앵커** | 중앙 (Node2D 원점) | Mirror > RotationPivot의 회전 중심과 일치 |
| **빔(부채꼴) 캔버스** | 160×160 | 128px 반경 + 여백. 중심을 기준으로 오른쪽 방향 부채꼴 |
| **하이라이트 테두리** | 48×64 | 본체보다 약간 큰 테두리. 선택 상태 전용 |

### 크기 비교 참고

```
  휴식 마커 (64x64)  ≒  거울 (64x64 캔버스 / 24x40 실효)  <  돌기둥 적 (64x128)
         1 타일                    0.4x0.6 타일                      1x2 타일
```

> 거울은 작고 정적인 오브젝트이지만, RotationPivot에 의해 4방향(90° 간격) 회전이 적용된다.
> 본체 스프라이트는 "위를 향한 기본 자세(0°)" 1장만 제작하고, Godot의 rotation으로 90°/180°/270°를 처리한다.

## 2. 디자인 컨셉

### 전체 느낌
- **"시간이 멈춘 숲에 남겨진 오래된 손거울/수정 조각"**
- 깨끗한 현대적 거울이 아니라, 숲에 묻혀 있던 고대 유물 같은 느낌
- 표면은 맑은 반사면 — "빛을 모아 어둠에 흘려보내는 장치"라는 설정과 연결
- 프레임/가장자리는 약간의 이끼, 금속 산화, 마법적인 각인 허용

### 형태 가이드
- **본체**: 세로로 긴 마름모 형태 (현재 placeholder와 동일 실루엣 유지). 상단/하단이 뾰족한 다이아몬드
- **반사면 (중앙)**: 매끈한 하늘색/은색 그라데이션. 유리나 수정의 광택
- **프레임 (테두리)**: 1~2px 두께의 어두운 선 또는 금속 틴트
- **핸들/스탠드**: 불필요 — 공중에 떠 있거나 바닥에 꽂힌 연출(설치 로직과 무관)
- **각인(선택)**: 중앙에 작은 룬/별 모양 1개. 시간 테마 강조

### 방향성
- **기본 자세(0°)**: 부채꼴이 **오른쪽**을 향할 때의 외형을 기준으로 제작
- 상/하/좌 방향은 Godot의 `rotation` 프로퍼티로 90° 회전하여 표현
- 반사면에 방향성 빛줄기/하이라이트가 있다면 "오른쪽 방향"으로 제작

- **하이라이트**: 본체 프레임과 구분되는 밝은 노랑 — 플레이어가 "상호작용 가능"을 즉시 인지

## 4. 필요 스프라이트 목록

### 4.1 본체 스프라이트 (정적, 필수)

| 태그/파일 | 캔버스 | 프레임 수 | FPS | 루프 | 설명 |
|---|---|---|---|---|---|
| `idle` | 64×64 | 1 | — | — | 기본 정지 자세. 반사면에 미세한 광택 고정 |
| `idle_anim`(선택) | 64×64 | 4 | 6 | O | 반사면 광택이 천천히 움직이는 루프. 미세한 "살아있는 거울" 연출 |

> **필수**는 `idle` 1프레임. `idle_anim`은 폴리싱 단계에서 추가 권장.
> Godot rotation으로 4방향을 처리하므로, 본체 스프라이트는 0°(오른쪽 반사 기준) 1장만 필요.

### 4.2 빔(부채꼴) 이펙트 (정적 또는 루프)

| 태그/파일 | 캔버스 | 프레임 수 | FPS | 루프 | 설명 |
|---|---|---|---|---|---|
| `beam_stopped` | 160×160 | 1 (또는 4) | 8 | O | STOPPED 중 표시되는 예측 부채꼴. 60° 각도, 반경 128px. 반투명 노란빛 |
| `beam_flowing` | 160×160 | 4 | 12 | X | FLOWING 진입 순간 0.3초간 번쩍이는 부채꼴. 선명도 상승 후 페이드 아웃 |

> 부채꼴 각도(60°)와 반경(128px)은 코드에 고정. 스프라이트는 **오른쪽 방향 부채꼴 기준**으로 제작.
> 본체와 함께 RotationPivot으로 회전되므로, 본체와 빔은 동일 방향 기준으로 정렬.
> 현재 코드는 Polygon2D로 동작하므로, 스프라이트가 없을 때는 기존 Polygon2D fallback 유지.

### 4.3 분열 트리거 순간 이펙트 (비루프)

| 태그/파일 | 캔버스 | 프레임 수 | FPS | 루프 | 설명 |
|---|---|---|---|---|---|
| `fx_mirror_flash` | 96×96 | 6 | 20 | X | FLOWING 진입 시 거울 본체 위치에서 번쩍. 빛 확산 → 수축 |
| `fx_shatter_burst` | 128×128 | 6 | 18 | X | 분열이 실제로 발생한 적 위치에 표시되는 파편 버스트 (1체 적중당 1회) |

> `fx_mirror_flash`는 거울 1개당 1회. `fx_shatter_burst`는 분열된 적 각각에 표시.
> 두 이펙트 모두 **없어도 동작**하며, 없을 시 코드에서 단색 원형 Tween으로 fallback.

### 4.4 하이라이트 테두리 (선택 상태 표시)

| 태그/파일 | 캔버스 | 프레임 수 | FPS | 루프 | 설명 |
|---|---|---|---|---|---|
| `highlight_border` | 48×64 | 1 (또는 4) | 8 | O | 본체 주변 노란 테두리. 정적 1프레임 또는 맥동 루프 |

> 현재 Line2D로 동작. 스프라이트 교체 시 NinePatchRect 또는 Sprite2D로 전환 가능.
> 4프레임 맥동 루프(알파 0.6 → 1.0 → 0.6) 권장.

### 4.5 상호작용 프롬프트 아이콘

| 태그/파일 | 캔버스 | 프레임 수 | FPS | 루프 | 설명 |
|---|---|---|---|---|---|
| `prompt_key_e` | 24×24 | 1 | — | — | [E] 키 아이콘. 프롬프트 Label 옆에 표시 |

> 범용 자산 — 다른 상호작용 오브젝트(거점, 상점 등)에서도 공유 권장.
> 없을 시 현재의 "[E] 거울 회전" 텍스트 Label로 fallback.

## 5. 애니메이션 상세 가이드

### `idle_anim` (선택, 4프레임 루프)
- 프레임 1: 반사면 중앙 광택 한 줄 (상단 위치)
- 프레임 2: 광택이 반사면 중간으로 이동
- 프레임 3: 광택이 반사면 하단으로 이동
- 프레임 4: 광택이 사라진 순수 반사면 상태
- 8fps 정도로 느리게 — "천천히 빛을 모으는" 느낌

### `beam_flowing` (4프레임 비루프)
- 프레임 1: 부채꼴 알파 0.35, 외곽 진한 테두리 발생
- 프레임 2: 알파 0.5, 가장 선명 (히트 프레임)
- 프레임 3: 알파 0.3, 페이드 시작
- 프레임 4: 알파 0.1, 거의 소멸
- 0.3초 내외. 실제 분열은 FLOWING 진입 순간 즉시 처리되므로 애니메이션은 "잔향" 역할

### `fx_mirror_flash` (6프레임 비루프)
- 프레임 1~2: 거울 중앙에서 작은 흰점 → 중간 크기 원형
- 프레임 3: 최대 크기 (96×96 전체 채움)
- 프레임 4~6: 페이드 아웃, 가장자리만 남김
- 총 0.3초. `beam_flowing`과 동시 재생

### `fx_shatter_burst` (6프레임 비루프)
- 프레임 1: 중앙에 작은 흰 플래시
- 프레임 2~3: 파편이 8방향으로 방사, 최대 확산
- 프레임 4~5: 파편이 흩어지며 희미해짐
- 프레임 6: 거의 소멸, 희미한 보라빛 잔상
- 색감: 흰 → 노랑 → 보라 (빛이 어둠에 반응하는 느낌)

### `highlight_border` (선택, 4프레임 루프)
- 프레임 1: 알파 0.6
- 프레임 2: 알파 0.8
- 프레임 3: 알파 1.0
- 프레임 4: 알파 0.8
- 8fps — 부드러운 맥동

## 6. Aseprite 작업 가이드

### 파일 구성

**본체(권장)**: 단일 `.aseprite` 파일로 통합 관리

```
obj_mirror.aseprite
[Tags]
├── idle       (프레임 1, 단일 프레임이면 태그 생략 가능)
└── idle_anim  (프레임 2~5, 선택)

총 1~5프레임, 캔버스 64x64
```

**빔 이펙트**: 별도 `.aseprite`

```
fx_mirror_beam.aseprite
[Tags]
├── stopped   (프레임 1)
└── flowing   (프레임 2~5, 4프레임 비루프)

캔버스 160x160
```

**기타 이펙트**: 각각 별도 `.aseprite`

```
fx_mirror_flash.aseprite   → 96x96, 6프레임
fx_shatter_burst.aseprite  → 128x128, 6프레임
```

### 내보내기 설정

**본체 스프라이트**:
```
파일 → Export Sprite Sheet
  → Layout: Horizontal Strip
  → Output: assets/sprites/objects/mirror/obj_mirror.png
  → 단일 프레임일 경우 단순 Save As PNG도 가능
```

**빔 이펙트**:
```
파일 → Export Sprite Sheet
  → Layout: Horizontal Strip
  → Tag: stopped
  → Output: assets/sprites/objects/mirror/fx_mirror_beam_stopped.png  (160x160)

파일 → Export Sprite Sheet
  → Layout: Horizontal Strip
  → Tag: flowing
  → Output: assets/sprites/objects/mirror/fx_mirror_beam_flowing.png  (640x160, 4프레임)
```

**분열 이펙트**:
```
fx_mirror_flash.aseprite   → fx_mirror_flash.png    (576x96, 6프레임)
fx_shatter_burst.aseprite  → fx_shatter_burst.png   (768x128, 6프레임)
```

**하이라이트 / 프롬프트**:
```
highlight_border.aseprite  → highlight_border.png   (48x64, 단일 또는 192x64 4프레임)
prompt_key_e.aseprite      → prompt_key_e.png       (24x24, 단일)
```

### 레이어 구성 권장 (본체)

```
[Layers]
├── shine     (반사면 광택 — 애니메이션 시 별도 레이어)
├── glass     (반사면 메인 색상)
├── frame     (마름모 프레임/테두리)
└── rune      (중앙 각인, 선택)
```

## 9. 우선순위

1. **본체 스프라이트** (`obj_mirror.png`) — 가장 먼저 눈에 띄는 핵심 오브젝트
2. **빔(정지)** (`fx_mirror_beam_stopped.png`) — STOPPED 상태에서 항상 보이는 예측선. 플레이어가 회전 각도를 이해하는 핵심 단서
3. **거울 발동 플래시** (`fx_mirror_flash.png`) — FLOWING 진입 순간의 임팩트
4. **분열 버스트** (`fx_shatter_burst.png`) — 적 분열 순간의 피드백. 폴리싱
5. **빔(흐름)** (`fx_mirror_beam_flowing.png`) — 부채꼴 잔향. 폴리싱
6. **하이라이트 테두리** (`highlight_border.png`) — 현재 Line2D로 충분히 동작
7. **E키 프롬프트** (`prompt_key_e.png`) — 범용 자산. 거울 전용 아님

> 현재는 모든 비주얼이 코드 fallback (Polygon2D, Line2D, Label)으로 동작한다.
> **본체 + 빔(정지)** 2장만 있어도 체감 개선 효과가 가장 크다.
> 분열 연출(3, 4번)은 메카닉의 핵심 피드백이므로 폴리싱 1순위로 권장.
