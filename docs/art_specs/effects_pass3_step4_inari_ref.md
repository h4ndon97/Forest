## 1. 선호 포인트 (사용자 확인)

- **전투 이펙트의 셰이더/발광 느낌** ← 가장 마음에 든 요소
- **배경 광원 처리** (이미지 2) — 전반적 색감이 아니라 **광원·쉐이더 연출**에 한정
- 장르·스토리·수리검 메카닉은 참고 대상 **아님**

## 2. 이미지 1 (전투 적중 GIF) — 훔칠 기법

| 요소 | 관찰 | Godot 4 재현 방법 |
|---|---|---|
| HDR 블룸 코어 | 중심은 거의 순백, 할로만 고채도 컬러 | `WorldEnvironment.glow_enabled` + **코어/할로 레이어 분리** (additive blend) |
| 방사형 라인 버스트 | 중심에서 방사되는 밝은 스트라이프 | 각도 기반 스트라이프 + radial fade **쉐이더 1장** |
| 화면 폭 단선 광선 | 얇은 흰 코어 + 두꺼운 컬러 글로우 | 흰 코어 `Line2D` + 컬러 글로우 `Line2D` **2중 레이어** |
| 백색 파티클 스파크 | 작고 고휘도, 짧은 수명 | 기존 피격 파티클 계열 확장 |

**레이어 분리의 중요성**: 코어(순백)와 할로(고채도)를 분리해야 "빛이 뜨겁다"는 인상이 난다. 한 스프라이트에 그라데이션 찍으면 흐릿해짐.

## 3. 이미지 2 (배경 광원) — 훔칠 기법 (Pass 3 범위 밖, 메모용)

- **거리별 대기 원근**: 원경 실루엣 → 중경 점광원 블룸 → 근경 선명. `WorldEnvironment.adjustment` + `Light2D` 점광 배치.
- **실루엣 + 림라이트**: 전경 건축은 거의 검정, 하늘 광원으로 외곽 엣지만 물듦. 2D에서는 하늘 레이어 + 전경 저명도 단색으로 구현.
- **공기 파티클**: 바람에 날리는 잔광. 기존 숨결 파티클 계열 확장.

→ **숲 버전 번역**: 달빛 갓레이, 멀리 거점 모닥불 점광원 블룸, 안개층, 전경 고사목 실루엣.

**주의**: 이 배경 기법은 **1구역 스프라이트 작업 단계** 이후에 다룸. Pass 3 Step 4 범위에 포함시키지 말 것.

## 4. Pass 3 Step 4에 직접 적용되는 결론

"시간 정지 해제 시점" = **파란 별버스트 + 파란 방사선**
→ INARI 이미지 1의 빨간 버전과 **구조 동일**, 컬러만 블루 계열로 스왑.

### 구현 순서 제안

1. `WorldEnvironment.glow_enabled` 베이스라인 (전역 bloom 약하게)
2. 블루 코어+할로 2레이어 머티리얼 (additive blend)
3. 방사선 burst 쉐이더 1장
4. 해제 시점 트리거 — EventBus `time_resumed` (또는 동등 시그널) 구독, 지속 0.15–0.25초

---

## 5. 주의사항

- 블룸·additive는 눈 피로도 높음. [보수적 파티클 초기값 원칙](../../memory/feedback_particle_initial_values.md) 그대로 적용.
- **항상 켜두지 말 것** — "비트"(시간 정지 해제 / 피니시 / 크리티컬 적중)에만 터뜨림.
- 숲의 정적 톤이 베이스. 이 이펙트는 대비로 존재해야 의미 있음.
- INARI는 "항상 화려한 화면"이 코어 — 네 게임은 정반대 방향이므로 **밀도만 가져오고 빈도·지속은 대폭 낮춤**.

---

## 6. 참고 가능한 추가 작업 (선택)

- 같은 구조를 **빛 공격 적중**에도 재사용: 컬러만 금색/백색으로 스왑. 머티리얼·쉐이더 공용화.
- **잔상 처리**: INARI 순간이동 잔상처럼, 시간 정지 중 이동 시 플레이어 위치 몇 프레임 남기는 트릭. 동일한 additive 블룸 톤으로 통일.
