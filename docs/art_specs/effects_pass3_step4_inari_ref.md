## 3. INARI 기법 분해 (13개)

Pass 5 플래닝과 이후 이펙트 작업 시 참조.

**전투/발광 (이미지 1)**
- **E1-1** 코어/할로 2레이어 블룸 — 순백 코어 + 고채도 할로 분리
- **E1-2** 방사형 라인 버스트 — 각도 기반 스트라이프 + radial fade
- **E1-3** 화면 폭 단선 광선 — 흰 코어 `Line2D` + 컬러 글로우 `Line2D` 2중
- **E1-4** 스캐터 스파크 — 작고 고휘도 파티클, 단발
- **E1-5** HDR 글로벌 블룸 베이스라인 — 모든 발광의 기반

**배경/대기 (이미지 2)**
- **E2-1** 거리별 대기 원근 — 원경 desaturate + 중경 점광원 bloom
- **E2-2** 실루엣 + 림라이트 — 전경 검정 + 하늘 광원 엣지 물듦
- **E2-3** 공기 파티클 — 바람 잔광, 약블룸 방울형
- **E2-4** 씬 컬러 그레이딩 — 씬 전체 톤 통일

**모션/전투 (트레일러 일반 특성)**
- **E3-1** 잔상 체인 — 여러 프레임 페이드 (✅ `EffectsAfterimage`로 구현됨)
- **E3-2** 무기 트레일 + 글로우
- **E3-3** 임팩트 프리즈 + 방사형 버스트 콤보

**레이어 분리의 중요성**: 코어(순백)와 할로(고채도)를 분리해야 "빛이 뜨겁다"는 인상이 난다. 한 스프라이트에 그라데이션 찍으면 흐릿해짐.

## 5. Pass 5 진입 시 권장 순서

1. **E1-5 글로벌 블룸 베이스라인 선행** — 1구역 아트 스프라이트 확정 후 값 튜닝. 다른 모든 발광 이펙트의 인프라. 보수적 초기값 `glow_intensity 0.3`, `glow_bloom 0.1`, `glow_hdr_threshold 1.0`, `glow_levels 2~4`부터 시작.
2. **E1-1 2레이어 블룸 리팩** — 기존 힛 플래시 피니시를 코어/할로 분리로 업그레이드. E1-5 깔려 있으면 비용 낮음.
3. **E1-2 방사형 burst 쉐이더** — 시간 정지 해제 강화 + 피니시 블로우 + 보스 페이즈 전환 공용.
4. **E3-3 프리즈 + 버스트 콤보** — 피니시 연출 고도화. E1-2 선행 필요.
5. **E2-3 공기 파티클** — 앰비언트. INARI식 약블룸 방울 버전.

Pass 4(땅거미)는 별도 경로. **E2-2 림라이트**를 근접 단계에 연동 (원거리 = 순수 실루엣 유지).

## 6. 주의점 / 충돌

1. **글로벌 블룸의 전파 효과** — `WorldEnvironment.glow`는 모든 밝은 픽셀에 작용. 기존 Light2D 등불·거울 룩이 변함. Pass 5 선행에서 이 튜닝이 가장 비용 큰 작업.
2. **픽셀아트 × 블룸 긴장관계** — intensity 보수적으로 시작. 코어 선명 유지. 공격적으로 올리면 픽셀 에지가 무너짐.
3. **세피아와 펄스의 레이어 순서** — 세피아 weight 0 Tween과 펄스 트리거 **동시 시작**. 이 원칙은 Step 4 구현에 이미 반영됨. Pass 5 방사형 burst 투입 시에도 동일 원칙 유지.
4. **INARI "요란함"의 역설** — 항상 화려한 게 INARI 코어. 우리는 반대. 블룸·additive의 **빈도**와 **지속 시간**을 비트에만.
5. **땅거미 실루엣 업그레이드의 위험** — 림라이트 전역이면 "어둠이 숨김" 톤 약화. 근접 단계에서만 활성화.
6. **보수 원칙 재확인** — [보수적 파티클 초기값 원칙](../../memory/feedback_particle_initial_values.md) 그대로 적용. 숲의 정적 톤이 베이스, 이펙트는 대비로 존재해야 의미 있음.

## 7. 배경 광원(이미지 2) 참고

장기 — 1구역 스프라이트·배경 작업 단계에서 재참조.

- **거리별 대기 원근**: 원경 실루엣 → 중경 점광원 bloom → 근경 선명
- **실루엣 + 림라이트**: 전경 건축 검정 + 하늘 광원 엣지 물듦. 숲 버전: 달빛 갓레이, 멀리 거점 모닥불 점광, 안개층, 전경 고사목 실루엣
- **공기 파티클**: 바람 잔광. 기존 숨결 파티클 계열 확장
