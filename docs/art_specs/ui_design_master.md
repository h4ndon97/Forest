### 2-1. 주톤 / 팔레트 방향

**선택**: **하이브리드 + 숲 자연 재질 중심**
- **인게임**: 다크 베이스(Ender Lilies 톤) + 이끼/덩굴/나무결 액센트
- **메타 UI**: 낡은 **나무판** 프레임 (양피지는 지도·대화창 내부 등 보조로만)
- "일반 RPG 양피지"가 아니라 **"숲에 녹아든 UI"** — 재질에 숲 DNA 직접 반영

**팔레트**:
| 역할 | HEX | 용도 |
|---|---|---|
| 시간 테마 골드 | `#F2CC66` | 시간 정지 UI, 스킬 준비 |
| 그림자 테마 보라 | `#8B2FC6` | 그림자 속성, 땅거미 |
| 빛 테마 화이트/크림 | `#F0E8D8` | 기본 텍스트 |
| **이끼 녹색 (1차 액센트)** | `#4A5A30` | 프레임 장식, 스킬 준비 완료, 환경 오브젝트 외곽 |
| **새순 연녹 (2차 액센트)** | `#6B8F4A` | 체력 회복, 세이브/거점, 긍정 피드백 |
| 배경 다크 브라운 | `#1A1410` | 인게임 HUD 배경 |
| 나무판 브라운 | `#5A4A30` | 메타 UI 프레임 |
| 경고/위험 | `#C84A4A` | 피격, 위험, 잠금 |

### 2-2. 스타일 톤

**선택**: **미니멀 + 자연 장식**
- 내부 레이아웃/정보 표시는 미니멀 (Ender Lilies / Death's Door 참조)
- 프레임 모서리는 나뭇가지/잎사귀/덩굴 장식 (9-slice 코너)
- 과한 장식 금지 — 전투 시야 방해 없음

### 2-3. 애니메이션 밀도

**선택**: **맥락 기반 + 자연 모션**
- **시간 정지(STOPPED)**: 극도로 정적 — 펄스도 멈춤
- **시간 흐름(FLOWING)**: UI 주변 포자/반딧불/낙엽 살아남, 잎사귀가 살짝 흔들림
- **맥락-메카닉 연동**: UI 상태가 게임 메카닉과 같은 언어를 말함

### 2-4. 팔레트 확장 — "숲의 기호학" 모티프 + 플레이어 기호학

**이중 축 구조** (중요):
- **세계/스테이지 UI** (환경 오브젝트, 월드맵, 메타 UI 배경, 트랜지션 등) → **숲 모티프 우선**
- **플레이어 HUD** (HP, 콤보, 스킬, 소모품, 성장 UI 등) → **빛/그림자 모티프 우선**, 숲 모티프는 **약하게 허용**

**Why**: 플레이어는 "숲에 들어간 빛·그림자를 다루는 자"라는 정체성. 세계(숲)와 플레이어(빛/그림자)가 시각적으로 대비되어야 함. 다만 금지까지는 아님 — 덩굴 구분선/반딧불 호버 등 약한 사용은 플레이어 UI에서도 유지.

**세계 UI 숲 모티프**:
| 모티프 | 주 사용처 | 대체 기호 |
|---|---|---|
| **잎사귀** | 체크마크, 월드맵 노드 아이콘 일부 | 꽃잎 |
| **덩굴** | 월드맵 연결선, 프레임 구분선 | 뿌리 |
| **나뭇가지** | 9-slice 코너, 메뉴 화살표 | 잔가지 |
| **포자** | 배경 앰비언트, 바이옴 표시 | 먼지 |
| **나이테 / 이끼 얼룩** | 버튼 배경 텍스처, 슬롯 내부 | 돌 이끼 |
| **낙엽** | 씬 트랜지션, 페이드 | 꽃잎 디졸브 |

**플레이어 HUD 빛/그림자 모티프**:
| 모티프 | 주 사용처 | 속성 연결 |
|---|---|---|
| **횃불 불씨** | HP pip (꺼지는 불씨) | 생명 = 지켜내야 할 빛 |
| **태양 (빛줄기)** | 콤보 속성 예고 (light) | 금색 `#F2CC66` |
| **반달/초승** | 콤보 속성 예고 (shadow) | 보라 `#8B2FC6` |
| **일식 (태양+달 겹침)** | 콤보 속성 예고 (hybrid) | 금↔보라 pulse |
| **잿빛 구슬 (dim orb)** | 콤보 속성 예고 (neutral) | 무채색 `#8A8A90` |
| **반딧불** | 호버 피드백, 시간 상태 인디케이터 | 시간 메카닉과 연결 (2-12) |

**공유 (양쪽 모두 사용)**:
- 덩굴: 플레이어 HUD의 스킬 슬롯 연결선(약하게), 세계 UI의 주 구분선
- 반딧불: 시간 상태(2-12) 역할 — 플레이어·세계 경계에 위치

→ 플레이어가 화면을 보자마자 **"나는 빛/그림자를 쥐고 숲에 들어온 자"**라는 정체성을 읽게 됨.

### 2-6. HUD 고유성 — 횃불 불씨 pip (잠정 확정 — 2026-04-18 개정)

체력 pip을 **"횃불 불씨(flame)"**로 표현. 체력 감소 시 불씨가 **사그라들어 꺼지는 모션**으로 상실 시각화.

**Why**:
- 플레이어 정체성(빛·그림자를 다루는 자) → HP = "지켜내야 할 내면의 빛"
- 숲(세계) 모티프인 잎사귀와 차별화 (2-4 이중 축 구조)
- 콤보 인디케이터의 원형 오브(태양/반달/일식/잿빛 구슬)와 **모양 분리** (원 ≠ 불꽃) — 시각 충돌 회피
- 꺼지는 불씨는 긴장감을 직관적으로 전달 — "위험" 상태 연출과 자연스러움

**규격 확정**:
- **5 pip × 5단계** (100% / 75% / 50% / 25% / 0%) = **5HP 단위로 까임**
- 최대 HP 100 기준 1 pip = 20 HP, 1단계 = 5 HP
- 10 HP 데미지 → 반 불씨 꺼짐 (사용자 의도 반영)
- 총 프레임 = 1불씨 × 5상태 = 5프레임 + 불꽃 흔들림 루프 (2~3프레임)

**연출**:
- 정상 상태: 불꽃 미세 흔들림(2~3프레임 루프) — 살아있음 표현
- 단계 전환 시 불씨 축소 + 연기 1~2파티클
- 0 도달 시 불씨 완전 소멸 + 잿가루 파편 낙하
- 회복 시 불꽃 확장 애니메이션 + 금색 `#F2CC66` 플래시
- 위험(20% 이하 = 1불씨 이하) 시 불꽃 거칠게 펄럭임 + 화면 크랙 오버레이 점등

**색 정책**:
- 기본 불꽃: 금빛 `#F2CC66` (빛 계열 중립)
- 시간 정지(STOPPED) 중: 반딧불 색 규칙(2-12)에 따라 시안 `#A8D8D0`로 변조 가능 — 플레이어도 정지됨을 표현
- 잿가루: 무채색 `#8A8A90` — 죽음/소멸 상태

**대안 (UI 가변성 원칙, 아트 작업 중 변경 가능)**:
- 횃불 대신 **광구(light orb) pip** — 태양/반달과 동일 원형 체계로 통일
- 빛·그림자 **이중 심볼** pip (반반 불씨) — hybrid 스타일 플레이어 강조

### 2-10. 시간 조작 메카닉과 자연 파티클의 규칙

- **STOPPED (시간 정지)**: UI 주변 잎/반딧불/포자 **모두 정지**. 전역 정지 메카닉이 UI 레이어까지 전파
- **FLOWING (시간 흐름)**: 모든 자연 파티클 정상 동작
- **MANIPULATING (조작 중)**: 슬로우 모션 (velocity × 0.3)

이 규칙은 메카닉과 UI의 시각 언어를 통일한다.

### 2-11. 검증된 정합성

### 2-12. 반딧불 색상 — 혼합안 (확정)

UI 호버·선택 강조에 쓰이는 반딧불은 **시간 상태에 따라 색이 변함**:

| 시간 상태 | 반딧불 색 | 의미 |
|---|---|---|
| **FLOWING (시간 흐름)** | 금색 `#D9B36A` | 숲의 생명, 따뜻함 |
| **STOPPED (시간 정지)** | 시안 `#A8D8D0` | 시간의 영적, 차가움 |
| **MANIPULATING (조작 중)** | 보간 `금 ↔ 시안` | 전환 중 (셰이더 1줄) |

**Why**: 플레이어가 글자/아이콘 없이 **주변 반딧불 색만으로 시간 상태를 즉각 인지**. UI 미니멀 + 메카닉 직결.
**구현**: 반딧불 스프라이트 1세트(흰색 단색 기준) + 코드 modulate. 프레임 증가 없음.

### 2-13. 낙엽 트랜지션 (확정)

**기본 전환**: 모든 씬 전환에 적용. 잎 20~40장, 1~2초.

**강화 연출 (상황별)**:
| 상황 | 연출 | 길이 |
|---|---|---|
| 보스 진입 | 붉게 물든 잎 + 느린 낙하 | 3~4초 |

**규격**: 기본 잎 스프라이트 3~4종 변형 + modulate로 붉음/재 변주. 상세는 `ART_RESOURCE_LIST.md` Phase 3-7 Pass 5b 반영 예정.

#### B-5. 해·달 궤도 시계 확정 (2026-04-18, 잠정, 규격 재확장 2026-04-23) — B1 + B3 통합

**결정**: 시간 상태 표시(B1)와 낮/밤 시각 인디케이터(B3)를 **해·달 궤도 시계 하나로 통합**. 원형 64×64 심볼이 두 역할 동시 수행.

**Why**:
- 해(빛)·달(그림자) = 플레이어 정체성 팔레트 직접 계승 (2-4 이중 축)
- 궤도 회전이 시간 상태 3종을 **직관적**으로 표현 (정상/정지/역회전)
- 시간 메카닉 = "빛과 그림자의 순환"이라는 게임 철학 시각화
- HUD 요소 1개 절약 — 우상단 복잡도 최소화

**규격 확장 경위**:
- **1차 확장 (2026-04-22, 48×48 / 16×16)**: 초안 32×32 + 해·달 8×8이 픽셀 버짓 과소. "HUD 2× 밀도" 산업 트릭은 640×360×3 세팅에서 정수 비율 부재 + snap_2d 설정으로 불가 확인 → virtual 픽셀 확대로 대체.
- **2차 확장 (2026-04-23, 64×64 / 32×32)**: 1차 결과도 실제 배치 시 **핵심 기믹 UI로서 비중 부족** 확인. 시간 조작은 게임 핵심 메카닉이므로 HUD 비중 상향이 정당.
- 최종 픽셀 버짓: **ring +300% / sun·moon +1500%** (원본 대비). 해/달에 이목구비·발광 단계·텍스처 충분히 표현 가능.

**구조 (64×64)**:

**낮/밤 전환 연출** (B3 통합):

| **3홉 이상 (≤12.5%)** | 보라 도트 4개 ●●●● (50% dim) | `#8B2FC6` dim | "거의 멈춘 상태" |

**표시 규칙**:
- flow 세션 활성 여부로 먼저 분기: **활성 아니면 배지 숨김**
- 활성 + 현재 = 원점: **금 도트 1개** (플레이어가 원점 위치 명시 인지)
- 활성 + 현재 = 전파됨: 홉 수에 해당하는 **보라 도트**
- STOPPED 상태: 배지 유지(스테이지의 **고유 rate**는 상태 무관)
- MANIPULATING: 배지 유지(별도 표기 없음)

**배치**:
```
B-5/B-6 시간 코어 (40×40) 하단에 접합 — 여백 확보
배지 중심: x=608, y=62 (기준 해상도 640×360)
좌상단: (603, 57) ~ (613, 67)
시간 코어 하단(y=52)과 배지 상단(y=57) 사이 여백 5px
```

**등장/퇴장 연출**:
- flow 시작 시 (원점 결정 순간): 금 도트 1개 페이드 인 + 짧은 빛 퍼짐
- 전파된 스테이지 진입 시: 배지 페이드 인 + 도트 순차 점등(+0.1초)
- flow 중단 (모든 스테이지 stop) 시: 배지 페이드 아웃
- 원점 변경 시 (stop 후 재시작, 다른 스테이지가 원점 됨): 금→보라 도트 전이 애니메이션(짧은 교차 페이드)

**이중 축 준수**: 원점=금(빛), 전파=보라(그림자 희석) → 플레이어 정체성 팔레트 유지. 숲 모티프 0.

**B-7b. 월드맵 전파 시각화 (전략 판단)**

F 카테고리에서 세부 확정. B-7에서는 **원칙만 명시**:

- **원점 노드**: 금빛 펄스(강, 0.8초 주기) + 큰 반딧불 궤적 1~2마리
- **전파 노드**: rate 비례 금빛 펄스(약, 1.2초 주기)
  - 50% 노드: 반딧불 1마리, 미세 호흡
  - 25% 이하: 잔잔한 미광만
- **정지 스테이지**: 차분한 잿빛 정지 상태
- **연결선 흐름 방향**: 원점 → 인접으로 느린 반딧불 입자 흐름(0.3초당 1개)
- **현재 플레이어 위치**: 별도 마커(F 카테고리 확정)

**숲 모티프 우선** (이중 축): 월드맵은 세계 UI → 반딧불·나이테·덩굴 허용. 플레이어 HUD 배지(빛/그림자)와 **축이 대비**되어 자연스럽게 "외부 세계 정보"로 읽힘.

**설계 의도**:
- 전역 원칙(섹션 2)이 모든 카테고리의 팔레트/톤을 통제
- 인게임 UI(A~D)는 "즉각적 정보 전달" 최우선
- 메타 UI(G)는 "세계관 몰입" 최우선
- 공통 하위(E~I)는 양쪽에서 공유되며 양쪽 원칙 모두 존중

## 5. 논의 및 확정 로드맵

각 카테고리는 아래 4단계로 논의·확정한다.

1. **현재 상태 확인** (본 문서에 이미 기재)
2. **레퍼런스 리서치 참조** (`references/ui_reference_analysis.md` 완료 후)
3. **사용자와 컨셉 방향 확정** (대화 기반)
4. **본 문서 + 개별 명세 동시 갱신** — 모순 없이

### 5-1. 진행 순서 (제안)

다음 순서로 카테고리별 논의 진행:
1. **섹션 2 전역 원칙** — 팔레트/스타일 톤 먼저 확정해야 하위 카테고리 결정 가능
2. **A. 인게임 HUD** — 플레이 시간 중 가장 자주 보는 UI
3. **B. 시간 시스템 UI** — 게임의 핵심 메카닉 시각화
4. **C. 공포·경고 UI** — 땅거미 연출은 게임 정체성
5. **H. 피드백·트랜지션** — 전투감에 직결
6. **D. 월드 인터랙션 UI** — 환경 오브젝트 활용 체감
7. **E. 인벤토리/장비/스킬** — 메타 진행감
8. **F. 월드맵** — 탐험감
9. **G. 메타 UI** — 게임 외곽 (세이브/설정/로딩)
10. **I. 버튼 프롬프트·커서** — 마지막 폴리싱

### 5-2. 문서 관계 규칙

- **본 문서(상위)**: 컨셉/연출 방향, 카테고리 간 관계, 전역 원칙
- **개별 명세(하위)**: 에셋 규격, 폴더, 파일명, 구현 세부
- **모순 발생 시**: 본 문서의 컨셉이 우선. 개별 명세가 먼저 쓰였더라도 본 문서 확정 후 재동기화
- **갱신 순서**: 사용자와 합의 → 본 문서 먼저 갱신 → 개별 명세에 "상세 참조" 링크 추가 → 개별 명세 세부 수정
