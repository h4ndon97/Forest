# 시간이 멈춘 숲 — 땅거미 아트 명세서

> 땅거미(Dusk Spider)는 시간이 흐르는 스테이지를 감지하고 추적해 오는 특수 적이다.
> 레벨 디자인 시 배치되지 않으며, 동적으로 스폰되어 맵에 등장한다.
> 일반 적보다 크고 강력한 위협으로, 시각적으로도 확실한 위압감이 필요하다.
>
> **상위 문서**: [ui_design_master.md](ui_design_master.md) 섹션 C (공포·경고 UI) 참조.
> 땅거미 HUD/비네트/경고 색상은 상위 문서의 C 카테고리에서 통합 결정됨.

---

## 1. 캔버스 / 크기 규격

| 항목 | 값 | 비고 |
|---|---|---|
| **캔버스 크기** | 192x192 | 일반 적(64~128)보다 큰 특수 적. 다리/촉수 모션 여백 확보 |
| **실효 크기** | 약 128x128 | 캔버스 내에서 본체가 차지하는 영역 (중앙 정렬) |
| **타일 대비** | 약 2x2 타일 (128x128) | 일반 적(1~1.5 타일)보다 확실히 큰 존재감 |
| **앵커** | 하단 중앙 | 다리가 바닥에 닿는 기준점 |

### 크기 비교 참고

```
  꽃 적 (64x64)  <  바위 적 (96x96)  <  나무 적 (128x128)  <  땅거미 (192x192 캔버스)
     1 타일              1.5 타일             2 타일              2~3 타일 (캔버스)
```

> 땅거미는 일반 적 중 가장 큰 나무 적보다도 큰 캔버스를 사용한다.
> 192x192 캔버스 안에서 긴 다리와 촉수가 움직이는 공간을 확보한다.

## 2. 캐릭터 디자인 컨셉

### 전체 느낌
- **"어둠에서 기어 나온 거대한 거미"** — 그림자 자체가 형태를 이룬 존재
- 완전한 거미가 아니라, 그림자로 이루어진 **왜곡된 거미 실루엣**
- 몸체는 반투명/연기 같은 질감, 다리는 날카롭고 각진 그림자 조각
- 눈(복안)은 희미하게 보라빛/붉은빛으로 빛남 — 유일한 밝은 포인트

### 형태 가이드
- **본체**: 둥글고 납작한 거미 몸통. 경계가 불명확하고 그림자가 피어오르는 느낌
- **다리**: 6~8개. 가늘고 날카로운 그림자 선. 끝이 뾰족하게 갈라짐
- **눈**: 4~6개의 작은 빛점이 이마 부근에 배치. 어둠 속에서 빛남
- **꼬리/후부**: 그림자 안개가 뒤로 흩어지는 느낌 (이동 시 잔상 효과)

### 핵심 차별점 (일반 그림자 적 vs 땅거미)
- 일반 적: 오브젝트(나무, 바위 등)의 형태가 왜곡된 모습
- 땅거미: **독립적인 거미 형태** — 특정 오브젝트에서 유래하지 않은 순수한 어둠의 존재

## 3. 색상 팔레트

### 메인 팔레트

| 용도 | 색상 코드 | 설명 |
|---|---|---|
| 본체 (어둠) | `#0A0A12` | 거의 순수한 검정에 가까운 남색. 그림자 본체 |
| 본체 (중간톤) | `#151525` | 본체의 볼륨/음영 표현 |
| 본체 (하이라이트) | `#1E1E35` | 빛이 닿는 본체 가장자리 |
| 다리 (기본) | `#0D0D1A` | 다리/촉수의 메인 색상 |
| 다리 (끝부분) | `#1A0028` | 다리 끝 → 보라빛으로 변하며 사라짐 |
| 눈 (기본) | `#8B2FC6` | 보라빛 발광. 유일한 밝은 포인트 |
| 눈 (밝음) | `#B84FE8` | 공격/활성 시 눈이 밝아짐 |
| 눈 (분노) | `#CC3333` | 공격 모션 시 붉은 빛으로 전환 (선택) |
| 그림자 안개 | `#0D0015`, 투명도 40~60% | 본체 주변에 피어오르는 연기/안개 |

### 색상 원칙
- 전체적으로 **극도로 어두운 색감** — 일반 적보다 한 단계 더 어두움
- 보라빛은 "그림자 에너지"를 상징 — 다리 끝, 눈, 안개에만 제한적 사용
- 빨간빛은 **공격/위협 상태**에서만 사용 (idle에서는 보라빛)
- 반투명 영역이 많음 — 본체 가장자리, 다리 끝, 안개

## 4. 애니메이션 태그 목록

### 스프라이트 시트 구성

| 태그명 | 프레임 수 | FPS | 루프 | 설명 |
|---|---|---|---|---|
| `idle` | 6 | 8 | O | 대기. 본체가 미세하게 맥동, 그림자 안개 흔들림, 다리 미세 움직임 |
| `crawl` | 6 | 8 | O | 이동. 다리가 교차하며 기어가는 모션. 그림자 잔상이 뒤에 남는 느낌 |
| `emerge` | 8 | 10 | X | **스폰 등장**. 바닥의 그림자에서 솟아오르는 연출. 검은 웅덩이 → 다리 출현 → 본체 부상 |
| `attack` | 6 | 10 | X | 공격. 앞다리 2개로 내려찍기/찌르기. 눈이 붉게 빛남 |
| `hurt` | 3 | 10 | X | 피격. 본체가 잠시 흩어졌다 재결합. 밝은 보라빛 플래시 |
| `dead` | 8 | 8 | X | 사망. 다리부터 그림자 입자로 분해 → 본체 소멸 → 보라빛 섬광 잔류 |

### 애니메이션 상세 가이드

#### `idle` (6프레임, 루프)
- 프레임 1~3: 본체가 살짝 위로 팽창
- 프레임 4~6: 본체가 살짝 아래로 수축
- 다리는 미세하게 위치 변동 (살아있는 느낌)
- 그림자 안개가 좌우로 흔들림

#### `crawl` (6프레임, 루프)
- 거미 특유의 다리 교차 보행 (좌1-우2-좌3-우4 순차)
- 본체는 전진하며 약간 위아래 흔들림
- 뒤쪽 다리 끝에서 그림자 잔상(안개)이 짧게 남음
- `idle`보다 전체적으로 약간 앞으로 기울어진 자세

#### `emerge` (8프레임, 비루프) **중요**
- 프레임 1~2: 바닥에 어두운 웅덩이(원형 그림자)가 나타남
- 프레임 3~4: 웅덩이에서 다리가 하나씩 솟아 나옴 (2~4개)
- 프레임 5~6: 나머지 다리 + 본체가 올라옴
- 프레임 7~8: 완전히 올라선 자세. 눈이 점등
- 도착 이벤트(`EventBus.dusk_spider_arrived`) 시 재생. 매우 인상적이어야 함

#### `attack` (6프레임, 비루프)
- 프레임 1~2: 앞다리를 높이 들어올림, 눈이 붉게 전환
- 프레임 3~4: 앞다리를 빠르게 내려찍기 (히트 프레임)
- 프레임 5~6: 다리 복귀, 눈 색상 보라로 복귀
- 히트 활성: 프레임 3~4 (0.2초, 10fps 기준)

#### `hurt` (3프레임, 비루프)
- 프레임 1: 피격 지점에서 보라빛 플래시 + 본체 일부 흩어짐
- 프레임 2: 흩어진 그림자 입자
- 프레임 3: 재결합하며 원래 형태로 복귀
- 전체 0.3초

#### `dead` (8프레임, 비루프) **중요**
- 프레임 1~2: 본체에 균열 발생, 보라빛 빛이 새어 나옴
- 프레임 3~4: 다리가 하나씩 그림자 입자로 분해
- 프레임 5~6: 본체가 팽창하며 불안정하게 흔들림
- 프레임 7: 본체 폭발적 분해 → 그림자 입자 사방 흩어짐
- 프레임 8: 작은 보라빛 섬광만 남음 (잔류 마커 전환 힌트)
- emerge의 역순 느낌이면서 더 극적

## 5. 특수 이펙트 (별도 스프라이트)

| # | 사용처 | 규격 | 파일명 | 설명 |
|---|---|---|---|---|
| 1 | 접근 경고 이펙트 | 64x64, 4프레임 | `fx_spider_warning.png` | 땅거미 접근 시 화면 가장자리에 표시. 어두운 그림자 파동/거미줄 무늬 |
| 2 | 등장 웅덩이 | 128x64, 4프레임 | `fx_spider_pool.png` | `emerge` 시 바닥에 깔리는 그림자 웅덩이. 맥동하는 원형 |
| 3 | 사망 잔류 | 64x64, 단일 프레임 | `fx_spider_residue.png` | 처치 후 남는 큰 그림자 얼룩. 일반 잔류 마커(32x8)보다 큼 |

### 이펙트 가이드

**접근 경고 (`fx_spider_warning.png`)**:
- 땅거미가 2맵 이내로 접근하면 HUD에 표시
- 어두운 보라/검정 색상, 맥동하는 느낌
- 4프레임 루프, 가로 스트립 (256x64)
- 거리가 가까울수록 코드에서 투명도/크기를 증가시킴

**등장 웅덩이 (`fx_spider_pool.png`)**:
- `emerge` 애니메이션과 동기화
- 바닥에 깔리는 2D 원형 그림자
- 4프레임: 작은 점 → 확장 → 맥동 → 최대 크기
- 색상: `#0D0015` 기반, 투명도 70~90%

**사망 잔류 (`fx_spider_residue.png`)**:
- 일반 적의 잔류 마커(`shadow_residue.png`, 32x8)보다 훨씬 큼
- 큰 보라빛 웅덩이 형태
- 단일 프레임 (코드에서 투명도 맥동)

## 6. Aseprite 작업 가이드

### 파일 구성

하나의 Aseprite 파일 (`dusk_spider.aseprite`)에서 모든 애니메이션 관리:

```
[Tags]
├── idle    (프레임 1~6)
├── crawl   (프레임 7~12)
├── emerge  (프레임 13~20)
├── attack  (프레임 21~26)
├── hurt    (프레임 27~29)
└── dead    (프레임 30~37)

총 37프레임, 캔버스 192x192
```

### 내보내기 설정

**메인 스프라이트 시트**:
```
파일 → Export Sprite Sheet
  → Layout: Horizontal Strip
  → 모든 태그를 하나의 시트로 통합
  → Canvas Size: 192x192 per frame
  → Output: assets/sprites/enemies/dusk_spider.png
  → 결과: 7104x192 PNG (192 x 37프레임)
```

**또는 태그별 별도 내보내기** (코드 구조에 따라):
```
파일 → Export Sprite Sheet
  → Tag 선택: idle / crawl / emerge / attack / hurt / dead (각각)
  → Layout: Horizontal Strip
  → Output:
      assets/sprites/enemies/dusk_spider_idle.png     (1152x192, 6프레임)
      assets/sprites/enemies/dusk_spider_crawl.png    (1152x192, 6프레임)
      assets/sprites/enemies/dusk_spider_emerge.png   (1536x192, 8프레임)
      assets/sprites/enemies/dusk_spider_attack.png   (1152x192, 6프레임)
      assets/sprites/enemies/dusk_spider_hurt.png     ( 576x192, 3프레임)
      assets/sprites/enemies/dusk_spider_dead.png     (1536x192, 8프레임)
```

**이펙트 파일** (별도 `.aseprite`):
```
fx_spider_warning.aseprite  → fx_spider_warning.png  (256x64)
fx_spider_pool.aseprite     → fx_spider_pool.png     (512x64)
fx_spider_residue.aseprite  → fx_spider_residue.png  (64x64, 단일)
```

### 레이어 구성 권장

```
[Layers]
├── eyes        (눈 발광 — 별도 레이어로 깜빡임/색변환 용이)
├── body        (본체)
├── legs        (다리)
├── shadow_fog  (본체 주변 안개/연기 효과, 투명도 높음)
└── ground      (바닥 그림자/웅덩이, emerge용)
```

## 9. 우선순위

1. **메인 스프라이트 시트** (`dusk_spider.png`) — 핵심. emerge + idle + attack 우선
2. **접근 경고 이펙트** (`fx_spider_warning.png`) — 긴박감 연출의 핵심
3. **등장 웅덩이** (`fx_spider_pool.png`) — emerge 연출 보조
4. **사망 잔류** (`fx_spider_residue.png`) — 일반 잔류 마커로 fallback 가능

> 현재는 모든 비주얼이 코드 fallback (ColorRect)으로 동작.
> 메인 스프라이트 시트만 있으면 가장 큰 시각적 개선 효과.
