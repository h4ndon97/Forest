# 키맵 (KEYMAP)

> **시간이 멈춘 숲** — 키보드 입력 매핑 한눈에 보기.
> 최종 결정: 안 A (모던 PC 표준) — Dead Cells × Ori 하이브리드.
> 기준 파일: [project.godot](../project.godot) `[input]` 섹션.

## 2. 액션 매핑 (분류별)

| 액션 | 주 키 | 보조 키 | 비고 |
|---|---|---|---|
| `move_up` | **W** | ↑ | 이중 바인딩 |
| `move_down` | **S** | ↓ | |
| `move_left` | **A** | ← | |
| `move_right` | **D** | → | |
| `jump` | **Space** | — | 좌엄지 |
| `dash` | **Left Shift** | — | 좌새끼 |

### 2-2. 시간·환경 시스템

| 액션 | 키 | 입력 방식 | 비고 |
|---|---|---|---|

## 3. 손 분포 다이어그램

```
좌손 (이동·시스템·소모)              우손 (전투)
─────────────────────────            ─────────────────────────
새끼: Tab Shift Ctrl                  새끼: L           ← skill_4
약지: Q  A     1                      약지: K           ← skill_3
중지: W  S     2                      중지: I  ← skill_2
검지: E  D  F                         검지: U  J ← skill_1, attack
엄지: Space                           엄지: (휴지)
```

**원칙**
- 좌손 = 시간/공간 통제 (이동·점프·대시·시간 정지·등불)
- 우손 = 행동 (공격·스킬 콤보)
- 양손 모두 홈 포지션 유지 가능. Hollow Knight식 한 손 클러스터 회피.

## 4. 핵심 콤보 운지 점검

| 콤보 | 좌손 | 우손 | 가능? |
|---|---|---|---|
| 이동 + 점프 | A/D + Space | — | ✅ |
| 이동 + 점프 + 공격 | A/D + Space | J | ✅ |
| **시간 정지 + 이동 + 공격** | Ctrl + A/D | J | ✅ (Ctrl=새끼-아래, 공간 충분) |
| 대시 + 공격 | Shift | J | ✅ (대시 직후 공격) |
| 점프 + 스킬 | Space | U/I/K/L | ✅ |
| **시간 정지 + 등불 토글** | Ctrl + Q | — | ⚠️ Ctrl(새끼) + Q(약지) — 가능하지만 약간 부담. 대안 검토 가능 |
| 환경 조작 + 점프 | E + Space | — | ✅ |

## 5. 설계 근거 요약

| 결정 | 근거 (레퍼런스) |
|---|---|
| WASD + 화살표 이중 바인딩 | Dead Cells, Ori — 가장 욕먹지 않는 패턴 |
| Space=점프, Shift=대시, Tab=인벤, Esc=폴, M=맵 | 9개 레퍼런스 중 8/9개 표준 |
| Time Manipulate = **Left Ctrl** (모달 키) | Katana Zero — 이동·공격과 동시 가능한 thumb-adjacent 모달 |
| Lantern = **Q** (전투 클러스터에서 분리) | 시스템 토글은 전투 키와 분리 — Blasphemous Q(Prayer) 참고 |
| 4 스킬 = **U I K L** (양손 분산은 안 했지만 우손 홈 포지션 정리) | Hollow Knight식 단일 행 산개 회피, J 옆 K로 콤보 친화 |
| Interact 분리 (F/E) | F=NPC/획득(Dead Cells F=Heal과 충돌 없음), E=환경(Ori 컨벤션) |
