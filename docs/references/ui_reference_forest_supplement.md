# 1. Ori and the Blind Forest / Will of the Wisps

## 1-1. 프로젝트 기본 정보
- 개발: Moon Studios, 2015 (Blind Forest) / 2020 (Will of the Wisps)
- 아트 스타일: 손그림 수채 + 빛 기반 후처리 (픽셀아트 아님)
- UI/아트 담당: Anna Jasinski (Ability Tree, Inventory, Skill Icons, Map Icons, HUD)
- **본 보충 리서치의 기준점**: "숲 UI의 표준"으로 평가받는 이유 — 빛(spirit)과 어둠(숲), 유기물 선형(덩굴·나뭇가지), 부드러운 발광이 UI 전 영역에서 일관된 문법으로 작동한다.

## 1-2. 자연 모티프 목록 (Ori)
| 모티프 | 사용처 | 비고 |
|---|---|---|
| **나뭇가지·뿌리** | Ability Tree의 노드 연결선, 메뉴 좌우 장식 | 유기적으로 분기, 직선 없음 |
| **잎사귀** | HP 오브의 "하얀 나무" 아이콘, 장식 장식 | 심벌로 축약되어 있음 |
| **발광 반딧불·포자** | 호버/선택 피드백, 스피릿 라이트 수집, 메뉴 배경 파티클 | 청백/금색의 작은 점들 |
| **영혼(spirit)의 빛줄기** | 메뉴 전환, Ability Tree 잠금 해제 시 방사형 flash | Ori 고유 스타일 |
| **조상나무(Ancestral Tree)** | 능력 획득 컷신 배경, 세이브 포인트 | 거대한 실루엣 + 코어에서 빛 방사 |
| **물 반사·이슬** | 맵 UI의 은은한 반짝임 | 주로 Will of the Wisps 맵 |
| **양치식물 실루엣** | 메뉴 하단/테두리 파랄랙스 | 전경 블러 |

## 1-3. 적용 위치별 세부

### HUD
- **좌상단 HP**: 녹색 오르브 + 중앙에 하얀 "작은 나무" 실루엣 아이콘. 현재 HP 숫자만큼 나무가 있는 오르브가 가로로 배치.
- **Energy(에너지)**: HP 아래 파란 오르브, 같은 디자인 언어.
- **Spirit Light(경험치)**: 작은 숫자 + 금빛 작은 점 아이콘. 레벨업 시 Ori 중심에서 방사형으로 빛 폭발, HUD에도 금빛 파티클이 잠깐 흐름.
- **디자이너 기록**: Will of the Wisps 개발 시 HUD 스케치가 약 60개 제작되었고, 최종적으로 Blind Forest의 것과 유사한 안이 채택되었다. — 즉 "충분히 미니멀하되 자연 모티프는 아이콘 수준에서만" 유지하는 전략.

### Ability Tree(능력 트리)
- **구조**: 중앙에서 3개 브랜치(공격/스피릿/움직임)로 분기, 각 브랜치는 시안·보라·자주 계열로 색 구분.
- **시각**: 브랜치는 나뭇가지 실루엣. 노드는 원형(영혼/씨앗). **잠금 해제 시** 노드 주변에 빛이 퍼지면서 다음 노드로 "수액이 흐르듯" 가지가 뻗어감 — 단순한 state 전환이 아닌 "성장"의 은유.
- **요구 미충족 표시**: 능력 설명 아래 텍스트가 빨간색으로 바뀜(시스템적 명료성).

### 맵
- **Spirit Well 노드**: 영역마다 1개 존재, 파란 빛 오르브로 표시.
- **현재 위치**: Ori 실루엣 + 펄스.
- **미탐험**: 짙은 청록 안개, 탐험 시 "밝아지는" 파급 애니메이션.

### 트랜지션
- **세이브/부활**: Ori 본체에서 빛 방사 → 화면 전체 블룸 → 복귀. HUD는 페이드 없이 그대로.
- **컷신 진입**: 레터박스 대신 화면 상하에 양치식물 실루엣이 서서히 스며 들어옴 → 영화적 프레임.

- **딥 나이트 블루**: `#07224B` (배경/메뉴 베이스)
- **포레스트 미드나잇**: `#173D91`
- **스피릿 시안**: `#2DA2DC` (발광의 주색)
- **오로라 바이올렛**: `#5B2755` (능력 트리 보라 브랜치)
- **웜 엠버**: `#C15630` (스피릿 라이트, 경험치, 위험 강조)
- **페일 크림**: `#EAE4C6` 추정 (UI 텍스트)

## 1-5. 픽셀아트 번역 가능성
- **쉬움**:
  - HP/에너지 오르브(원+작은 나무 실루엣) → 32x32 가능, Ender Lilies 패턴과 유사
  - 금빛 파티클(레벨업) → 2x2 픽셀 점 8개 랜덤 스프레이로 충분
- **보통**:
  - Ability Tree 가지 뻗기 연출 → 픽셀로는 "노드 간 점선이 하나씩 켜지는" 방식으로 축약 가능
  - Spirit Well 펄스 → 3프레임 원형 확산
- **어려움**:
  - 부드러운 블룸/글로우의 누적 레이어 — 픽셀에서는 하드 엣지라 "빛의 밀도감"이 안 나옴. 대신 **2~3단계 색 밴드**로 구조적 발광을 시뮬레이션해야 함.
  - 수채 배경 파랄랙스 — 픽셀로는 실루엣 레이어 2장으로 치환 권장.

## 1-6. 솔로 개발자 적용 전략 (Ori)
- **차용**: HP 오브 + 나무 심벌, 능력 트리 가지 구조, 레벨업 파티클, 메뉴 배경의 떠다니는 반딧불/포자.
- **포기**: 수채 블룸, 실시간 고해상도 글로우, 고밀도 컷신 연출. "빛의 양감"은 픽셀에선 깎이므로 **심벌(나무·잎)과 동작(펄스·확산)**을 우선 차용.

# 2. Greak: Memories of Azur

## 2-1. 프로젝트 기본 정보
- 개발: Navegante Entertainment, 2021
- 아트 스타일: **손그림 2D 애니메이션** (픽셀이 아닌 셀 애니) — 디즈니 초기작 질감
- 핵심 시스템: 3형제(Greak / Adara / Raydel) 스위칭, 각자 HP/MP 개별 관리
- 본 보충 리서치의 의미: "손그림 자연 UI"가 어떻게 작동하는지, **픽셀로 치환 시 포기해야 할 부분**을 역산출

## 2-2. 자연 모티프 목록 (Greak)
| 모티프 | 사용처 | 비고 |
|---|---|---|
| **양피지·모험서** | 맵/퀘스트/기록 화면 | 책을 넘기는 형태의 메타 UI |
| **고대 석판 & 룬** | 스토리/로어 | 숲 + 고대 문명 혼합 |
| **꽃·꽃잎** | Adara(중간 자매) 관련 이펙트, 일부 장식 | 마법 속성 표현 |
| **물/강** | 3형제 중 Raydel(궁수) 및 물의 도시 | 청색 계열 UI 액센트 |
| **하늘색/에메랄드** | 캐릭터 3명의 HP 컬러 구분 | Greak=녹/에메랄드, Adara=보라-핑크, Raydel=청 (추정) |

## 2-3. 적용 위치별 세부
### HUD
- **3개의 HP/MP 바**: 화면 좌상단에 한 캐릭터 바만 강조, 나머지 두 캐릭터는 축약 아이콘으로 배경에 놓이는 구조로 알려져 있음.
- **현재 캐릭터 초상화**: 손그림 원형 프레임 + 이름 + HP/MP 수평 바.
- **마법 표식**: 마나 감소 시 잔잔한 이펙트 발산.

### 인벤토리
- **손그림 그리드 + 슬롯 제약**: **3개 시작 슬롯**만 제공, 퀘스트 아이템이 슬롯 소모. **리뷰에서 평이 나빴던 요소** — "과도한 제약이 탐험의 즐거움을 깎음". → 우리에게는 반면교사.
- **패치 후 UI 아트 대폭 개선**: 출시 초기와 이후 버전을 비교하면 "손그림 톤을 유지하면서 정보 밀도를 높이는" 작업이 이뤄짐.

### 맵
- **상세 맵 부재 → 오버월드 맵만 존재**: 지역 단위 이동/패스트 트래블 기반. 세부 맵은 없고, 나중에 "모험서"의 한 섹션으로 맵 페이지가 추가됨.
- **교훈**: 손그림 맵은 정보량 대비 제작비가 크므로, **지역 단위 노드 맵**이 솔로 개발자에게 유리. 이건 우리 프로젝트 방향과 정확히 일치.

### 트랜지션
- 캐릭터 스위치 시 페이드 + 줌 in/out. 자연 모티프 연출은 상대적으로 절제됨 (손그림 연출 자체가 본체 분위기를 지배).

- **세피아 양피지**: `#C9B48A` (UI 배경)
- **소프트 잉크**: `#3A2E24` (텍스트, 외곽선)
- **크림슨 레드**: `#A04040` (데미지, 위험)
- **페일 블루**: `#A8C8D8` (마법/물)

## 2-5. 픽셀아트 번역 가능성
- **쉬움**:
  - 양피지 인벤토리 배경 → 64x64 타일 + 잉크 얼룩으로 충분
  - 원형 캐릭터 초상 프레임 → 32x32 외곽선 링
- **보통**:
  - 손그림 외곽선의 "손맛" → 픽셀에서는 **1px 불규칙 노이즈 윤곽**으로 흉내
- **어려움**:
  - 셀 애니 특유의 **유기적 선의 두께 변화** — 픽셀에서는 거의 복원 불가, 포기 권장
  - 3캐릭터 동시 표시 HUD — 우리 프로젝트는 단일 주인공이므로 해당 없음

## 2-6. 솔로 개발자 적용 전략 (Greak)
- **차용**:
  - "모험서/책" 메타포의 메뉴 프레임 — 페이지 탭으로 맵/퀘스트/장비/코덱스 묶기
  - 양피지 배경 + 잉크 텍스트 콤보의 **정보 밀도 + 따뜻한 톤** 조합

  - 3슬롯 인벤토리의 제약 설계 (나쁜 UX 사례)

# 3. Hollow Knight — Greenpath 지역 전용 연출

## 3-1. 전제 확인
> **Hollow Knight는 "지역별 HUD 색 변화"를 하지 않는다.** HUD(마스크/소울 오르브/지오)는 전 지역에서 일관되게 유지된다.
> 대신, **배경/파티클/조명이 지역의 개성을 전담**한다. 이게 Greenpath(그린패스)를 숲 UI 분석 대상으로 삼는 이유.
> 우리 프로젝트도 이 전략(HUD는 통일, 지역은 배경/파티클로 차별화)을 참고하면 솔로 개발 부담이 크게 줄어든다.

## 3-2. Greenpath 자연 모티프 목록
| 모티프 | 사용처 | 비고 |
|---|---|---|
| **이끼 카모플라주** | 지역 이름 자체 ("Mosskin" 부족), 적 실루엣 | 색/질감이 배경과 거의 동화 |
| **넓은 잎** | 전경 파랄랙스, 떨어지는 낙엽 파티클 | 적이 이동할 때 잎이 흩날림 |
| **소용돌이 조각(Swirling carvings)** | 알 모양 석주, 고대 조형물 | 숲 + 고대 문명 혼합 |
| **산성 호수의 녹청** | 배경 반사 + 위험 지형 표식 | 밝은 시안-녹 |
| **스테인드글라스적 빛줄기** | 천장에서 내려오는 햇빛 축 | 게임 후반부 Stone Sanctuary는 더 어두운 대비 |
| **포자·먼지 파티클** | 앰비언트 공중 점 | 속도가 느리게 떠다님 |

## 3-3. Greenpath 팔레트 (리서치 기준)
- **Leaf Green**: `#4E7E3B`
- **Moss Green**: `#2F5D34`
- **Fern Green**: `#A0CFA3`
- **Earth Brown**: `#7C4C2C`
- **Sky Blue**: `#A4D4E1` (천장 개방부 하늘)
- 참고: 이 팔레트는 우리가 제안한 이끼 녹색 `#4A5A30`, 새순 `#6B8F4A`와 **거의 동일한 계열**. 방향 정합성 검증 완료.

## 3-4. 지역 개성을 담당하는 레이어 구성 (HK 패턴)
1. **전경 블러 파랄랙스**: 큰 양치식물 / 잎사귀 실루엣 (거의 검은색)
2. **중경 배경**: 실제 게임 공간의 타일맵 (이끼/돌/넝쿨)
3. **원경 파랄랙스 1**: 먼 숲 실루엣 (중간 녹)
4. **원경 파랄랙스 2**: 하늘/빛줄기 (옅은 시안)
5. **파티클 레이어**: 낙엽(피격/이동 시), 먼지(상시), 포자(선택적)

### UI 위에 얹히는 지역 특성
- **지역명 카드**: 지역 진입 시 세리프 타이포가 하단 중앙에 나타났다 사라짐. 색은 흰색이지만 배경이 녹색이라 자연스럽게 Greenpath 분위기가 배어 나옴.
- **벤치(세이브 포인트) 아이콘**: 지역 고유 장식은 없으나, 벤치 자체가 해당 지역 재질(돌·나무)로 지어져 있어 "UI=환경 일체화" 효과.

## 3-5. 적이 UI의 일부가 되는 사례
- **Moss Charger / Volatile Mosskin**: 이끼를 뒤집어쓴 형태 → 공격 전까지는 **배경과 거의 구분 불가**. 이건 UI 이벤트(경고 표식)를 **의도적으로 생략**하고 플레이어의 관찰력을 요구한 디자인 선택.
- **Lifeblood Cocoon**: 상호작용 가능한 파란 주머니 → 깨면 Lifeseed가 튀어나와 추격. "인터랙션 프롬프트 없음" — 반짝임만으로 표현. 우리 프로젝트의 "빛과 그림자" 톤과 궤를 같이 함.

## 3-6. 픽셀아트 번역 가능성
- **쉬움**:
  - 낙엽 파티클: 4~6프레임, 32x32 시트 안에 4가지 잎 모양 x 회전 2방향
  - 포자/먼지: 2x2 픽셀 점, 상향 이동 + 알파 페이드
  - 파랄랙스 실루엣: 큰 양치식물 1장 + 중간 나무 1장 + 먼 산 1장으로 충분
- **보통**:
  - 빛줄기(god ray): 픽셀에선 45도 대각선 반투명 기둥으로 치환. 애니메이션은 옵션.
- **어려움**:
  - 이끼 표면의 유기적 그라데이션 — 픽셀은 3~4색 밴드로 축약.

### F-01 잎사귀 HP pip — 시드는 잎 (상세)
- 규격: 16x16 또는 24x24

### F-02 덩굴 구분선 (상세)
- 규격: 1x8 픽셀을 수평 반복 타일링, 중앙에 작은 새순 심벌
- 바리에이션: 짧은 것(2타일), 긴 것(8타일), 섹션 타이틀용(가운데 잎 1개)

### F-04 반딧불 호버 피드백 (상세)
- 규격: 각 반딧불은 2x2 또는 3x3 픽셀
- 스폰: 호버 시작 시 아이콘 주위에 2~4마리 랜덤 스폰, 호버 유지 중 부유
- 애니메이션: 3프레임 발광 (밝음/중간/어두움) 루프
- 색: 새순 `#6B8F4A` 발광 → 금빛 `#D9B36A` 선택적

### F-05 포자 확산 선택 피드백 (상세)
- 선택 확정 순간 버튼에서 6~10개 포자가 외곽으로 방사형 퍼짐
- 수명 0.4초, 알파 페이드
- 규격: 2x2 픽셀, 회전 없음 (픽셀 회전은 픽셀 무결성 해침)

### F-08 낙엽 트랜지션 (상세)
- 씬 전환 시 화면에 20~30장의 낙엽이 회전하며 비스듬히 떨어짐 → 낙엽으로 화면이 거의 가려진 순간 씬 스왑 → 낙엽이 흩어지며 제거
- 규격: 낙엽 스프라이트는 4종류 x 4회전 프레임 = 16개
- 주의: 픽셀 회전은 "프리렌더 4~8방향"으로 대체 (실시간 회전 금지)

### F-09 수액 흐르기 해금 연출 (상세)
- 스킬 트리에서 노드 A → B로 해금 시, 둘을 잇는 가지 선이 **밑둥부터 점진적으로 밝아지며** B 노드에 도달

- 참고: Ori가 이걸로 "성장"의 은유를 만듦 — **우리 프로젝트의 성장 시스템과 직접 연결**

### F-11 양피지·모험서 메타 UI (상세)
- 메뉴/인벤토리/맵/코덱스를 한 "모험서" 책으로 묶음
- 페이지 탭 4~5개, 왼쪽 잎사귀 북마크 탭

### F-13 바람에 흔들리는 전경 잎 (상세)

# 7. 주의 사항 — "이런 건 픽셀에서는 잘 안 먹힘"

## 7-1. 하지 말아야 할 것
1. **실시간 픽셀 회전** — 낙엽·잎·반딧불 모두 프리렌더 방향 스프라이트로만.
2. **Ori식 블룸의 직접 모방** — 고해상도 블룸은 픽셀의 하드 엣지와 충돌. 3색 밴딩으로 구조적 발광을 흉내내는 것으로 충분.
3. **Greak식 유기 선 두께 변화** — 펜촉의 두께 변화는 픽셀 1px 경계에선 표현 불가. 외곽선은 1px 단색으로 통일.
4. **지역별 HUD 색 변화** — Hollow Knight조차 안 함. HUD는 전 지역 공통, 개성은 **배경/파티클**이 맡는다. (솔로 비용 절감)
5. **꽃잎의 3D 회전 / 나뭇잎의 섬세한 메쉬 휨** — 픽셀 8x8~16x16 잎에서는 입체감이 손실. "바람 방향 = 좌/우 기울임 2프레임"으로 평면적 표현이 정답.
6. **긴 책 넘김 애니메이션 (1초 이상)** — 자주 쓰이는 인터랙션은 0.3~0.5초, 자주 못 넘기면 스트레스.

## 7-2. 지켜야 할 것
- **일관된 팔레트 고정**: 메인 이끼 녹색 `#4A5A30`, 새순 액센트 `#6B8F4A`, 반딧불/스피릿 금색 `#D9B36A`, 텍스트 크림 `#EAE4C6` 축을 벗어나지 않기.
- **HUD는 미니멀, 지역은 배경으로 차별화**: HK Greenpath가 증명한 솔로 최적 전략.
- **"심벌 + 동작"이 픽셀의 정답**: Ori는 양감으로 표현하지만, 픽셀은 **특정 심벌(잎/가지/반딧불)**과 **특정 동작(펄스/확산/시듦)**의 조합으로 의미를 전달한다.

## 7-3. 엣지 케이스 — 시간 조작과의 상호작용
우리 프로젝트의 **시간 정지 중**에는 아래처럼 UI 자연 요소도 멈춰야 톤이 맞는다:
- 바람에 흔들리는 잎 → **정지** (프레임 고정)
- 반딧불 부유 → **정지**
- 포자 확산 → **정지 프레임 유지**
- HP 잎 pip 시듦 → 시간 정지와 무관하게 유지 (플레이어 상태이므로)

Katana Zero의 시간 슬로우가 "채도 감소 + 살짝 블러"로 분위기 전환했듯, 우리는 "자연 파티클 정지 + 약한 세피아 오버레이"로 시간 정지를 표현할 수 있다.

# 8. 문서 한계 및 결정 필요 사항

## 8-1. 한계
- Ori 원작 UI 원화는 Anna Jasinski 포트폴리오(ArtStation)에서 직접 확인이 필요하나, WebFetch 차단으로 텍스트 기반 기술로만 추정.
- Greak의 UI 개선 전/후 비교 스크린샷 확보 실패 — 개선 후 버전이 현재 상용이므로 상용 기준으로 서술.
- Hollow Knight Greenpath는 지역 자체는 잘 기술되었으나, "지역 진입 시 UI에 얹히는 연출" 관련 공식 정보가 빈약 — 커뮤니티 관찰 기반 추정 포함.

## 8-2. 사용자 결정 필요
1. **F-08 낙엽 트랜지션을 기본 전환으로 삼을 것인가, 특정 상황(지역 이동)에만 쓸 것인가?**
   - 기본 전환으로 쓰면 강한 정체성, 남용 리스크
2. **F-11 양피지·모험서 메타 UI를 채택할 경우, 기존 Phase 2-6 성장 시스템/Phase 2-7 인벤토리 UI와의 통합 시점은 언제인가?**
   - Phase 3-7 UI 일괄 적용 시 통합 권장
3. **HP 잎 pip(F-01)의 숫자는 몇 개인가?**
   - 5개(Ender Lilies 스타일) / 7~9개(Hollow Knight 스타일) 중 선택 필요
4. **반딧불 색은 금색(`#D9B36A`) vs 시안(`#A8D8D0`) 중 메인 팔레트에 어느 것?**
   - 금색: 따뜻함, 숲의 "생명" 강조
   - 시안: 차가움, 시간 조작의 "영적" 강조
   - 혼합: 일반 호버=금, 시간 정지 중=시안 (차별화 추천)
